# Contributing Guidelines

Thank you for your interest in contributing. This is a public-safe repository. Please read and follow these guidelines to keep contributions aligned with our guardrails.

What’s welcome
- Improvements to high-level documentation and diagrams
- Public-safe examples and templates without internal logic
- Fixes to clarity, tone, and accessibility

What’s out of scope
- Proprietary algorithms or orchestration logic
- Internal logs, credentials, infrastructure details, or private documents
- Code or docs copied from private repositories

Submitting changes
1. Open an issue describing your proposal.
2. Submit a PR referencing the issue. Keep changes focused and small.
3. Ensure you’ve reviewed `PUBLIC_REPO_POLICY.md` and `CODE_OF_CONDUCT.md`.

Style
- Keep it clear, concise, and public-safe. Prefer diagrams and examples over lengthy theory.
- Use placeholders for any credentials or endpoints.

License
- Please coordinate on repository license with maintainers before adding any new code artifacts.
